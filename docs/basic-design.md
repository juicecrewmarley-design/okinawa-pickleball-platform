# 基本設計

## 目的

沖縄県内のピックルボール人口を2030年までに1万人へ増やすため、会員管理、大会運営、OPRポイント、ランキング、イベント発信、協賛企業PRを一元化する公式Webアプリを構築します。

## 対象ユーザー

- 一般会員: 会員登録、会員証表示、大会エントリー、ランキング確認
- 大会参加者: 大会情報確認、カテゴリ選択、エントリー
- 運営管理者: 会員、大会、参加者、試合結果、OPR、お知らせ、協賛企業の管理
- 協賛企業: 企業情報掲載、PRリンク導線

## MVP方針

最初のMVPでは、大会運営に必要な最短導線を優先します。

- 会員登録とログイン
- QRコード付き会員証
- 大会一覧と大会エントリー
- 管理者画面の初期UI
- OPRランキング表示
- Googleフォーム既存会員IDの引き継ぎ
- ダブルス/チーム戦のメンバー紐づけ完了によるエントリー確定
- 会員/非会員別の参加費と、各種目ごとの定員設定

## データベース設計

主なテーブルは以下です。

- `profiles`: 会員プロフィール、会員ID、権限
- `legacy_members`: Googleフォーム登録済み会員、既存会員IDの引き継ぎ元
- `tournaments`: 大会情報
- `tournament_entries`: 大会エントリー
- `match_results`: 試合結果
- `opr_points`: OPRポイント履歴
- `notices`: お知らせ
- `sponsors`: 協賛企業
- `sponsor_contacts`: 協賛企業担当者

集計用に`annual_rankings`ビューを用意します。ランキングは年度、種目、総合で抽出しやすいよう、ポイント履歴を正規化して保存します。

## 権限設計

- 一般会員: 自分のプロフィール、エントリー、ポイントを閲覧できます。
- 管理者: 全会員、大会、参加者、結果、ポイント、お知らせ、協賛企業を管理できます。
- 未ログイン: 公開大会、お知らせ、協賛企業、ランキングを閲覧できます。

Supabase Authの`auth.users.id`と`profiles.id`を1対1で紐づけます。管理者判定は`profiles.role = 'admin'`で行います。

## 会員登録ルール

- 登録画面では「所属エリア」は入力しません。
- 居住地は「沖縄県内」または「沖縄県外」から選択します。
- 沖縄県内の場合は、沖縄県内41市町村から選択します。宮古島市、石垣市、竹富町、与那国町など離島も含みます。
- Googleフォーム登録済み会員は、フォームで発行された`OKP-0001`形式の会員IDを入力できます。メールアドレスと会員IDが`legacy_members`で一致した場合のみ、その番号を引き継ぎます。

## 大会カテゴリとエントリー確定ルール

管理者が大会作成時に選択できるカテゴリは以下です。

- ダブルス選択枠: 男子ダブルス、女子ダブルス、ミックスダブルス
- ダブルスカテゴリ: オープン初中級者、オープン中上級者、35+、50+、65+
- チーム戦カテゴリ: 合計年齢100+、150+、180+、200+

大会作成時は、選択した各種目ごとに定員を設定します。参加費は会員参加費と非会員参加費を分けて設定します。

エントリーは、必要なメンバー紐づけが完了した時点で確定します。

- ダブルス: 申込者本人 + ペア1名
- チーム戦: 申込者本人 + 追加メンバー3名、合計4名
- 必要人数がそろっていない場合は`pending`、そろった場合は`confirmed`です。
- 非会員もエントリーできます。非会員申込は連絡先と参加費を保存し、管理者確認後に確定できます。

## OPRランキング表示

- 年間OPRランキングは、男子ダブルス、女子ダブルス、ミックスダブルスから種目を選びます。
- 各種目のカテゴリは、オープン初中級者、オープン中上級者、35+、50+、65+です。
- 総合ランキングは男子総合と女子総合に分けて表示します。
- `opr_points`には年度、種目、カテゴリ、総合の性別、参加ポイント、勝利ポイント、順位ポイントを保存します。

## 将来のアプリ化を見据えた構成

- 画面ロジックをNext.js App Router配下に整理
- Supabase AuthとDBをAPI基盤として利用
- QR会員証データはWeb/モバイル共通で使えるJSONペイロードに統一
- UIはスマホ操作を優先したレスポンシブ設計
- 大会、ランキング、会員証は将来React NativeやExpoへ移植しやすい単位で分離

## 開発手順

1. Supabaseプロジェクトを作成
2. `supabase/schema.sql`をSQLエディタで実行
3. `.env.local`にSupabase URLとAnon Keyを設定
4. Next.jsアプリを起動
5. 会員登録、ログイン、大会エントリーを確認
6. 管理者ユーザーの`profiles.role`を`admin`へ変更
7. 管理画面から大会、結果、OPR付与の運用画面を拡張
